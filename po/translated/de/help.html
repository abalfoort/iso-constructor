ISO Constructor Hilfe

ISO Constructor helps building your custom Debian ISO, and was originally created for maintaining SolydXK ISOs, but it might benefit those who want to build their custom Debian ISO.

1. Eine Distribution hinzufügen

Unpack an ISO

Die heruntergeladene ISO Datei finden und ein Verzeichns, in das die ISO Datei entpackt werden soll, aussuchen.

Die Schaltfläche zur Auswahl des Auspackzielverzeichnisses wird verfügbar, wenn der Pfad zur ISO Datei existiert.

Add an existing directory

Wenn Sie bereits eine ISO Datei entpackt und die Distribution aus der ISO Constructor Liste entfernt haben, können Sie das bestehende Verzeichnis hinzufügen, indem Sie den Dialog "Von der ISO Datei erstellen" abwählen.

Die Sichern Schaltfläche wird verfügbar, wenn sowohl das Zielverzeichnis der ISO Datei als auch das Auspackverzeichnis existieren.

2. Distributionen entfernen

Es werden nur die ausgewählten Distributionen entfernt. Das Distributionsverzeichnis und sein Inhalt wird NICHT gelöscht.

3. Verzeichnis öffnen

Öffnet das ausgewählte Arbeitsverzeichnis im Standard Dateimanager.

4. Im Terminal editieren

Öffne ein "chrooted" Terminal und beginne die Installation (und die Löschung) aller benötigten Pakete. Es bietet sich die Konfiguration im Wurzelverzeichnis der Distribution zu ändern und nicht im Terminal.

5. Aktualisieren von Distributionen

Führt einfach "apt-get dist-upgrade" aus, wobei jedoch zu berücksichtigen ist, dass einige Dienste vor und nach dem Upgrade behandelt werden müssen.

6. Erstelle die ISO Datei

Dafür sind folgende Schritte notwendig:

  • Laden Sie die offline Pakete herunter (/lib/live/mount/medium/offline).
  • Installieren Sie alle Firmwaretreiber.
  • Entfernen Sie verwaiste und nicht verfügbare Pakete (einschließlich des alten Kernels).
  • Konfigurieren Sie das System.

Der Erstellungsvorgang kann sehr lange dauern. Wenn Sie den Prozess beenden müssen, bevor er endet, können Sie ein Terminal öffnen und ausführen:

sudo killall mksquashfs

Für den Fall, dass Sie mehrere Distributionen aus der Liste ausgewählt haben, müssen Sie den Befehl wiederholen.

7. Lokalisierung der ISO Dateien

Damit wird Ihre ISO Datei lokalisiert, die damit verbunde Auswahl der Sprache und Zeitzone besteht aus 6 Schritten:

  1. Festlegung der gewünschten Sprache

  2. Festlegung der Standardsprache

  3. Festlegung der Zeitzone

  4. Änderung der live Konfiguration (/etc/live/config.conf).
  5. Führen Sie ein Upgrade durch (apt-get dist-upgrade).
  6. Installieren Sie bei Bedarf Pakete für KDE, Firefox, Thunderbird, LibreOffice und andere Lokalisierungspakete. Überprüfen Sie das setlocale.sh-Skript auf lokale-spezifische Pakete.

Vergessen Sie nicht, boot/boot/grub/grub/grub.cfg und boot/isolinux/isolinux/isolinux.cfg der ISO zu übersetzen.

Debian live als Basis verwenden

Sie können sogar eine Debian Live DVD als Basis verwenden: http://www.debian.org/CD/live Wenn Sie Debian verwenden, benötigen Sie das Verzeichnis "isolinux" (und seinen Inhalt), um das Verzeichnis "isolinux" im Debian-Auspackverzeichnis zu ersetzen. Danach müssen Sie die Distribution von Grund auf neu konfigurieren. Es ist eine Menge Arbeit, aber für eine CE (Community Edition) lohnt es sich.

9. Wichtige Dateien

  • ~/.constructor/distributions.list: Hier werden alle Distributionsverzeichnisse gespeichert.
  • /usr/share/iso-constructor/grubgen.sh: erzeugt boot/boot/grub/grub.cfg (with grub-template).
  • /usr/share/iso-constructor/isolinuxgen.sh: erzeugt boot/isolinux/isolinux.cfg (with isolinux-template).
  • /usr/share/iso-constructor/cleanup.sh: dieses Skript bereinigt vor dem Erstellen der ISO.
  • /usr/share/iso-constructor/setlocale.sh: setzt das Gebietsschema der ISO. Standard ist: en_US
  • /usr/share/iso-constructor/excludes: Liste mit Verzeichnissen, die vom ISO Build ausgeschlossen sind.
